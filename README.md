# Privacy Policy

Last Updated: September 1, 2025

## 1. Overview

Check Me ("we," "our," or "us") is committed to protecting your privacy. This Privacy Policy explains how we collect, use, and protect your information when you use our mobile application (the "App"). By using Check Me, you agree to the collection and use of information in accordance with this policy.

## 2. Information We Collect

### 2.1 Automatically Collected Information
- **Device Information**: Device model, OS version, app version, device identifier
- **Usage Statistics**: App usage frequency, feature usage patterns, interaction data
- **Analysis Results**: Condition scores (0-100), analysis timestamps, mood states (text only - NO facial images are collected or stored)

### 2.2 Optional Information
- **User Preferences**: Theme settings, notification preferences
- **User Profile**: Optional nickname or display name

### 2.3 Camera Data Processing
- **Real-time Processing Only**: Camera feed is processed in real-time for analysis
- **No Storage**: Facial images are NEVER saved to device storage or transmitted to servers
- **Immediate Deletion**: All image data is removed from memory immediately after analysis

## 3. How We Process Information

### 3.1 Processing Purpose
- Providing real-time condition analysis and feedback
- Tracking condition trends over time (using numerical data only)
- Improving app functionality and user experience
- Generating anonymized usage statistics

### 3.2 Processing Method
- **On-Device Processing**: All facial analysis occurs exclusively on your device
- **No Server Transmission**: Facial images and biometric data never leave your device
- **Real-Time Analysis**: Processing happens in real-time with immediate data disposal

### 3.3 Explicit Non-Retention of Face Data
**Check Me does not retain any face data. All facial analysis occurs in real-time, and both original images and biometric data are completely erased from memory immediately after processing. We do not store, transmit, or backup any facial images or biometric information.**

## 4. Data Storage and Retention

### 4.1 Locally Stored Data
- **User Settings**: Stored until app deletion
- **Theme Preferences**: Stored until app deletion
- **App Configuration**: Stored until app deletion

### 4.2 Analysis Results Storage (NOT Face Data)
**Face data itself is NEVER stored. Only the following numerical analysis results are retained:**
- **What We Store**: 
  - Condition scores (numerical values 0-100)
  - Analysis timestamps (date and time)
  - Mood states (text labels like "Happy", "Tired", "Focused")
- **Why We Store This**: 
  - To provide daily, weekly, and monthly condition tracking
  - To show trends and patterns in your wellbeing
  - To enable the history feature
- **Storage Duration**: 
  - Maximum 30 days (to provide meaningful history while minimizing data retention)
  - Older entries are automatically deleted on a rolling basis
- **User Control**: 
  - Users can manually delete any or all history entries at any time
  - All data is deleted when the app is uninstalled

## 5. Data Sharing and Third Parties

### 5.1 Information We DO NOT Share
- **Facial Images**: NEVER shared with anyone
- **Biometric Data**: NEVER collected or shared
- **Personal Identification**: No personally identifiable information is shared
- **Raw Analysis Data**: Individual analysis results are never shared

### 5.2 Anonymized Data We May Share
- **Usage Statistics**: App open frequency, feature usage counts (via Google Analytics)
- **Ad Performance**: Ad interaction metrics (via Google AdMob) - no personal data
- **Crash Reports**: Technical error data (via Firebase Crashlytics) - no personal data

### 5.3 Purpose of Limited Sharing
- Improving app stability and performance
- Understanding feature usage patterns
- Measuring advertising effectiveness
- Fixing bugs and technical issues

### 5.4 Third-Party Sharing Statement
**Check Me does not share any face data, facial images, or biometric information with third parties:**
- **Face Data Sharing**: NONE - No facial data ever leaves your device
- **Ad Networks**: Receive only anonymized usage statistics (absolutely no face data)
- **Analytics Services**: Firebase Analytics receives only aggregated, anonymous metrics like check counts and feature usage
- **Cloud Services**: We do not use cloud storage for any user data

## 6. Data Security

### 6.1 Technical Safeguards
- **Encryption**: Sensitive data is encrypted using industry-standard methods
- **Sandboxing**: Data is isolated within the app's secure container
- **Access Controls**: Data is protected by iOS security features
- **No Network Transmission**: Face data never travels over networks

### 6.2 Operational Safeguards
- **Minimal Data Collection**: We only collect what's necessary
- **Immediate Disposal**: Face data is deleted immediately after processing
- **Regular Security Updates**: We maintain current security patches
- **Privacy by Design**: Security is built into every feature

## 7. Your Rights and Controls

### 7.1 Access Rights
- View all your stored analysis results in the History section
- Export your data through the app's export feature

### 7.2 Deletion Rights
- Delete individual analysis results at any time
- Clear all history data with one action
- Complete data removal upon app uninstallation

### 7.3 Control Rights
- Opt-out of analytics collection in Settings
- Control notification preferences
- Manage data retention settings

### 7.4 Correction Rights
- Update your profile information at any time
- Modify your preferences and settings

## 8. Children's Privacy

Check Me does not knowingly collect personal information from children under 13 years of age (or under 16 in the European Economic Area). If we become aware that we have collected personal information from a child under the applicable age, we will take steps to delete such information.

## 9. International Data Transfers

All data processing occurs on your device. We do not transfer your data internationally as no personal data leaves your device.

## 10. California Privacy Rights (CCPA)

California residents have additional rights under the California Consumer Privacy Act (CCPA):
- Right to know what personal information is collected
- Right to know if personal information is sold or disclosed (we do NOT sell any data)
- Right to opt-out of sale of personal information (not applicable as we don't sell data)
- Right to delete personal information
- Right to non-discrimination for exercising privacy rights

## 11. European Privacy Rights (GDPR)

If you are in the European Economic Area, you have additional rights under the General Data Protection Regulation (GDPR):
- **Legal Basis**: We process data based on legitimate interests and consent
- **Data Portability**: Export your data in a machine-readable format
- **Right to Object**: Object to certain types of processing
- **Right to Restrict**: Request restriction of processing
- **Supervisory Authority**: Lodge a complaint with your local data protection authority
